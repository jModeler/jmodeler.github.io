Resolving Install Errors with NVlabs imaginaire

July 30, 2024 One-minute read
AI • Python • Package Install
Python

TL;DR: I need to install the NVlabs imaginaire package for a project (more details in a later post). I walk through the installation steps and how to resolve any errors you might encounter.

  1. I need to install the NVlabs imaginaire package, I follow the instructions here.

  2. As explained in a previous post, I use pyenv to handle various versions of python and I’m currently using version 3.10.5:

    $ pyenv versions
    system
  * 3.10.5 (set by /Users/<username>/.pyenv/version)
    3.11.4
    3.12.3
  1. First make sure you have the latest version of pip by running the following command in terminal:
    pip install --upgrade pip
  1. For installing the package, I’m using the following commands in terminal (as shared here):
    git clone https://github.com/nvlabs/imaginaire
    cd imaginaire
    bash scripts/install.sh
    bash scripts/test_training.sh
  1. I hit the following error:
    Traceback (most recent call last):
    File "/Users/<username>/imaginaire/imaginaire/model_utils/gancraft/voxlib/setup.py", line 6, in <module>
      from torch.utils.cpp_extension import BuildExtension, CUDAExtension
  ModuleNotFoundError: No module named 'torch'
  make: *** [all] Error 1
  1. It appears the package torch is missing. I use pip to install this package.
    pip install torch
  1. I try running bash scripts/install.sh again. NOTE. I have to install the pre-reqs (CUDA 11.1 and cudnn) and it looks like this will not work on MAC OSX.
